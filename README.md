# MCP Benchmark EC2 Medium

Tier-specific EC2 runtime for the medium-tier MCP benchmark servers.
Regenerated by `mcp-benchmark/scripts/sync_ec2_repos.py` from the registry +
`sam/profiles/tiered.json` (local cold-spawn latency re-tiering, 2026-05-28).

## Medium Tier Servers

1. `mcp-osint-server`
2. `mcp-hn`
3. `biomcp`
4. `mcp-nixos`
5. `game-trends-mcp`
6. `wikipedia-mcp`
7. `math-mcp`
8. `okx-mcp`
9. `car-price-mcp-main`
10. `osm-mcp`

## Key Files

- `ec2/medium-route-config.json` - route config for `/medium/mcp` and `/medium/health`.
- `ec2/install-medium.sh` - clones/builds the medium-tier servers (GENERATED).
- `ec2/bootstrap-medium-host.sh` - host deps + install + runtime build + PM2.
- `ec2/ecosystem.medium.config.cjs` - PM2 process def using the route config.
- `ec2/runtime/` - Fastify streamable-HTTP to stdio MCP bridge.
- `ec2/adapters/` - vendored stdio adapters for this tier (GENERATED).

## EC2 Setup

```bash
bash ec2/bootstrap-medium-host.sh
```

- MCP endpoint: `http://127.0.0.1:3000/medium/mcp`
- Health endpoint: `http://127.0.0.1:3000/medium/health`

`servers/` is empty in Git; the install flow populates it on the host at pinned commits.
